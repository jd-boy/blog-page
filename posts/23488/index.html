Nginx proxy_pass 转发规则

 2022-07-06 22:51:50

一、proxy_pass 只包含 HOST

例如：

  • http://host - √
  • https://host - √
  • http://host:port - √
  • https://host:port - √
  • http://host/ - x
  • http://host:port/ - x

这种情况下，会把匹配到的所有路径直接穿透转发。比如以下的配置

访问 http://127.0.0.1:80/api/test, 最终的请求地址是 http://127.0.0.1:3000/api/test

二、proxy_pass 包含路径

这里的路径哪怕只是一个 / 也是存在的，如：

  • http://host - x

  • http://host:port- x

  • https://host/ - √

  • https://host:port/ - √

  • http://host/api - √

  • http://host/api/ - √

这种情况下，url 里面会 去掉 location 匹配的字符串，拼接到 proxy_pass 再进行转发。

访问 http://127.0.0.1:80/api/test，最终的请求地址是 http://127.0.0.1:3000/test

三、重写代理链接 - url rewrite

使用 rewrite 指令并且生效后，proxy_pass url 链接中的路径会被忽略，如：

访问 http://127.0.0.1:83/bb/cc，最终的请求地址是 http://127.0.0.1:3000/node/bb/cc，因为匹配上 / 了，没有匹配 rewrite 。

访问 http://127.0.0.1:83/api/cc，最终的请求地址是 http://127.0.0.1:3000/fixpath=cc，我们写的 proxy_pass http://127.0.0.1:3000/node/ 里面的 node 路径丢失了。
