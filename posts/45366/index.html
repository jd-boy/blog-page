1.type

System

表数据已经被加载到内存中，不需要进行磁盘IO的SQL

const

  1. 命中主键或者唯一索引
  2. 判断条件为常量值

示例：
id 为主键

name 为唯一索引

eq_ref

对于前表的每一行，后表只有一行被扫描。即

  1. join查询
  2. 连接时命中 主键 或 非空唯一索引
  3. 等值连接

示例：
test 表为 eq_ref

ref

字段索引为 非唯一索引 ，即同一索引值可能扫描到多条记录。

示例：
name 为普通索引

range

索引上的范围查询，如 between、in、>、< 都是范围查询

示例：
name 为普通索引

id 为主键

index

需要扫描 索引 上的全部数据，例如 select count(*) from user;

all

没有索引或索引失效，全表扫描。

示例：
name 为普通索引

id 不是主键，无任何索引
