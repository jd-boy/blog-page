MySQL MVCC
JD Lv5
 2021-11-25 15:59:25  
  • MySQL 
 

Mysql 有 读未提交（RU）、读已提交（RC）、可重复读（RR）、序列化，四种隔离级别，MVCC可以作用在 读已提交 和 可重复读 两种隔离级别上，同时在 可重复读 级别上解决了幻读。

1.隐藏字段

在每一行数据中，会有三个隐藏字段 db_row_id、data_trx_id、data_roll_ptr。

  • db_row_id：在没有设置主键，没有唯一索引的情况下作为隐藏行号，实现索引的；

  • data_trx_id：最近一次对该行更新的事务的id；

  • data_roll_ptr：回滚指针，指向其上一个版本的 undo log。

当对一条数据更新后，会将其原本的数据生成一条 undo log，然后将 DATA_ROLL_PTR 指向该 undo log ，形成一条版本链，方便回滚和MVCC 查找可访问的数据版本。

2.ReadView

在 ReadView 中有以下四个主要属性：

  • mIds：生成 ReadView 时，所有正在活跃（未提交的）的事务id；
  • min_trx_id：mIds 中最小的事务id；
  • max_trx_id：生成 ReadView 时，最大的事务id，注意 不是正在活跃的最大事务id（或者说mIds中的最大id）。因为先提交的事务不一定先结束，可能当前最大事务id为100，而 mIds 中最大的事务id是98，也就是说id为98的事务还未提交，而id为99和100的事务已经提交了；
  • creator_trx_id：创建该 ReadView 的事务id。

举个栗子：

如果 mIds = {5, 6, 7, 11, 13}，那么 min_trx_id = 5，max_trx_id > 13 或 max_trx_id = 13。

在 RC 级别，MVCC 会在每次读时生成一个新的 ReadView；

在 RR 级别，MVCC 只会在第一次读时生成 ReadView，当前事务之后都使用这同一个 ReadView。

3.访问规则

  1. 如果 data_trx_id 小于 min_trx_id ，说明生成当前版本的事务在生成 ReadView 之前就已经提交了，那么当前事务可以访问该版本数据。
  2. 如果 data_trx_id 大于 max_trx_id 说明当前版本的数据是在生成 ReadView 之后才生成的，所以当前事务不可访问该版本数据。
  3. 如果 data_trx_id 在 mIds 中，即大于等于 min_trx_id 且小于等于 mIds 中的最大事务id（注意不是 max_trx_id），说明创建该 ReadView 时，生成该版本数据的事务还未提交，所以不能访问。
  4. 如果 data_trx_id 大于 mIds 中的最大事务id且小于等于 max_trx_id ，说明创建该 ReadView 时，生成该版本数据的事务已经提交，所以可以访问。

当发现当前版本数据无法访问时，可以通过 data_roll_ptr 对 undo log 进行遍历，根据以上规则找到可以访问的数据版本。
